# 🏅 Medallion ETL Pipeline — Sales & Forecast Analytics

> A production-grade **Bronze → Silver → Gold** data pipeline that transforms raw retail JSON data into a validated **Galaxy Schema** dataset, ready for Power BI dashboarding.

## 🗄️ Data Model (Galaxy Schema)

Two fact tables share a common set of dimension tables:

```
Dim_Brand ──────────────────────────────────────┐
     │                                           │
Dim_Customer ──┐                           Dim_Product
Dim_Date ──────┼──── Fact_Sales                 │
Dim_City ──────┤                           Dim_Brand
Dim_Country ───┘
                     Fact_Forecast_2009
                      ├── Dim_Brand
                      ├── Dim_Country
                      └── Dim_Date
```

| Table | Type | Rows | Primary Key |
|---|---|---|---|
| `Dim_Brand` | Dimension | 11 | `BrandKey` |
| `Dim_Country` | Dimension | 3 | `CountryKey` |
| `Dim_Customer` | Dimension | 8,868 | `CustomerKey` |
| `Dim_Product` | Dimension | 2,495 | `ProductKey` |
| `Dim_City` | Dimension | 306 | `CityKey` |
| `Dim_Date` | Dimension | 731 | `DateKey` |
| `Fact_Sales` | Fact | 298,246 | `SaleKey` |
| `Fact_Forecast_2009` | Fact | 33 | `ForecastKey` |

## ⚙️ ETL Pipeline — Key Logic

### Bronze Layer
- Loads `Sales.json` and `forecast.json` directly into Pandas DataFrames
- Reports schema, volume metrics, cardinality, and null census — **no transformations**

### Silver Layer
- Strips whitespace from all string columns
- Parses `OrderDate` (`MM/DD/YYYY`) into datetime; generates integer `DateKey` offset from anchor `2008-01-01`
- Casts types: `ProductKey`, `CustomerKey`, `Quantity`, `OrderDateKey` → `int32`; `Net Price` → `float64`
- Fills null `Name` → `"Customer_<key>"`, null `Education`/`Occupation` → `"Unknown"`
- Runs a full **Data Quality suite**: regex, range, enum, null, uniqueness, and IQR outlier checks
- Flags color anomalies (raw `Color` field equals `Subcategory` in some records)

### Gold Layer
- Builds all 6 dimension tables with sequential surrogate integer keys
- Derives `Customer_Type` (`registered` vs `guest`) from Education availability
- Re-derives product `Color` from the last word of `Product Name` against a 14-color palette
- Joins foreign keys across tables; validates all PK uniqueness and FK referential integrity
- Generates a gapless `Dim_Date` calendar spine from `2008-01-01` to `2009-12-31`
- Exports all 8 tables to CSV and performs round-trip row-count verification

## 🚀 Getting Started

### Prerequisites

```bash
pip install pandas
```

### Run the Pipeline

```bash
# 1. Ensure source data is in place
mkdir -p data
# Place Sales.json and forecast.json inside data/

# 2. Run the pipeline
python ETL_medallion_pipeline.py

# Output CSVs will be written to Fact__Dimensions/
```

### Open the Dashboard

Open `Sales-Overview.pbix` in **Power BI Desktop**. Refresh the data source to point to your local `Fact__Dimensions/` directory.

## 🔑 Key Assumptions

| # | Assumption |
|---|---|
| 1 | **DateKey anchor** is `2008-01-01 = 1`. DateKey 367 = `2009-01-01`. |
| 2 | **Forecast granularity** is annual only (Brand × Country × Year 2009). |
| 3 | **Customer_Type** = `guest` when Education is `Unknown`; `registered` otherwise. |
| 4 | **Product Color** is re-derived from the last word of Product Name (14-color vocabulary), not taken from the raw `Color` field which was found to be anomalous. |
| 5 | **No rows are deleted** for outliers; IQR outlier detection is for observability only. |
| 6 | **Surrogate keys** are sequential integers with no business meaning; they may differ across pipeline runs if source data changes. |
| 7 | **Dim_Date spine** is gapless from `2008-01-01` to `2009-12-31` (731 days). |
| 8 | The pipeline is **idempotent** — re-running it with the same inputs produces identical outputs. |
| 9 | **Countries in scope**: China, Germany, United States. |

## ✅ Data Quality Checks

The pipeline runs **automated DQ validation** at every layer:

| Check | Applied To |
|---|---|
| Regex pattern match | ProductKey, CustomerKey, dates, names, text fields |
| Range bounds | Keys, Quantity ≥ 1, Net Price ≥ 0, DateKey ∈ [1, 731] |
| Enum membership | Continent (7), Education (6), Occupation (6), Customer_Type (2), Color (15) |
| Null check | All key and required columns |
| Uniqueness | All primary keys |
| IQR outlier detection | Quantity, Net Price |
| Foreign key integrity | All 9 FK relationships in the Galaxy Schema |
| DateKey sequence gaps | Dim_Date must have no gaps |
